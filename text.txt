git init

وظيفته: بيحول أي فولدر عادي على جهازك لـ "مستودع Git" (يعني بيفتح "الخزينة" للبرنامج).

استخدامه: بتستخدمه مرة واحدة بس لما تبدأ أي مشروع جديد.


git status

وظيفته: بيوريك حالة الملفات (إيه اللي اتعدل، إيه اللي جاهز للرفع، وإيه اللي لسه الـ Git مش شايفه).

استخدامه: ده "نظارتك"، لازم تبص فيه قبل أي حركة عشان متغلطش.

git add .

وظيفته: بيجهز الملفات (بيحطها في الـ Staging Area) عشان تاخد "لقطة" (Commit) ليها.

استخدامه: قبل أي commit.

git commit -m "message"

وظيفته: بياخد "لقطة" (صورة) ثابتة لحالة الملفات اللي جهزتها.

استخدامه: لما تخلص جزء صغير من الكود وعايز تحفظه.

ثانياً: أوامر المراجعة والتاريخ

git log

وظيفته: بيعرضلك تاريخ المشروع (كل الـ commits اللي عملتها بالترتيب).

استخدامه: عشان تفتكر إيه اللي عملته قبل كدة.

git log --oneline

وظيفته: نفس الـ log بس بيختصر كل شيء في سطر واحد عشان الشاشة متزحمش.

استخدامه: لما تحب تاخد نظرة سريعة على التاريخ.

git diff

وظيفته: بيوريك "التغييرات" اللي عملتها في الملفات (إيه اللي تمسح وإيه اللي اتزود) قبل ما تعمل add.

استخدامه: عشان تراجع شغلك وتتأكد إنك معملتش تعديل غلط.

git diff --staged

وظيفته: بيوريك التغييرات اللي عملتلها add فعلاً ومستنية الـ commit.

ثالثاً: أوامر البرانشات (الفروع)

git branch

وظيفته: بيعرضلك كل البرانشات اللي عندك وبيوريك أنت واقف فين حالياً.

git branch -m master

وظيفته: تغيير اسم البرانش اللي أنت واقف عليه لاسم master.

git checkout -b branch-name

وظيفته: بينشئ برانش جديد وينقللك عليه فوراً.

استخدامه: لما تبدأ "ميزة" (Feature) جديدة في المشروع.

git checkout branch-name

وظيفته: التنقل بين البرانشات الموجودة مسبقاً.

git branch -d branch-name

وظيفته: حذف برانش (بعد ما تكون خلصت شغلك فيه ودمجته).

git merge branch-name

وظيفته: دمج شغل برانش معين في البرانش اللي أنت واقف عليه حالياً.

رابعاً: أوامر الطوارئ والرجوع

git reset --hard commit-hash

وظيفته: الرجوع في الزمن ومسح كل شيء بعد الـ commit ده (خطير).

استخدامه: لو عايز تدمر شغلك الأخير وترجع لحالة قديمة.

git revert commit-hash

وظيفته: إلغاء تأثير commit معين بس بدون مسح التاريخ (بيعمل commit عكسي).

استخدامه: عشان تصلح خطأ بأمان.

git reflog

وظيفته: سجل بكل حركاتك (حتى الحاجات اللي حذفتها بـ reset).

استخدامه: عشان تلاقي أي حاجة ضاعت منك.

خامساً: أوامر الربط بـ GitHub (المستودع البعيد)

git remote -v

وظيفته: بيوريك رابط المستودع اللي أنت مربوط بيه على النت.



git push -u origin master

وظيفته: رفع شغلك من جهازك لـ GitHub لأول مرة (وربط البرانشين ببعض).

git pull origin master

وظيفته: تحميل التحديثات من GitHub ودمجها مع شغلك في خطوة واحدة.

git fetch

وظيفته: تحميل التحديثات من GitHub "على جنب" من غير ما يدمجها (عشان تراجعها أولاً).
